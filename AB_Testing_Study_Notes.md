## 1. A/B Testing 基础

### 核心目的
> 用数据判断两个方案之间的差异是**真实效果**还是**随机波动**

### 场景例子
```
你把"立即购买"按钮从蓝色改成绿色：
A组（对照组）：蓝色按钮 → 点击率 5.0%
B组（实验组）：绿色按钮 → 点击率 5.8%

问题：0.8%的差距是真实效果还是随机波动？
→ 这就是 A/B Testing 要回答的！
```

## 2. 假设检验 H0 & H1

| | 定义 | 例子 |
|---|---|---|
| **H0（零假设）** | 默认立场，假设没有效果 | "绿色和蓝色按钮没有区别" |
| **H1（备择假设）** | 你想证明的 | "绿色按钮点击率更高" |

### 法庭比喻
```
H0 = 被告无罪（默认）
H1 = 被告有罪
实验 = 检察官收集证据

目标：收集足够证据推翻H0，支持H1
```

### ⚠️ 重要：结论的严谨表达

| p-value | 正确说法 | 错误说法 |
|---|---|---|
| p < 0.05 | ✅ "拒绝H0" | ❌ "接受H1" |
| p ≥ 0.05 | ✅ "不能拒绝H0" | ❌ "接受H0" / "H0是对的" |

> **"不能拒绝H0"≠"H0是正确的"**
> 就像法庭"证据不足无法定罪"≠"被告一定无罪"

## 3. p-value 深度理解

### 定义
> **p-value = 假设H0为真，出现当前结果或更极端的概率**

### 本质：面积问题！
```
H0分布（正态分布）：

        ████
      ████████
    ████████████
  ████████████████
████████████████████
←——正常范围（95%）——→  |右尾(5%)|

p-value = 右侧尾部面积
        = "当前结果在H0世界里有多罕见"
```

### 判断规则
```
p-value < α（0.05）→ 结果显著 → 拒绝H0 ✅
p-value ≥ α（0.05）→ 结果不显著 → 不能拒绝H0 ❌
```

### 用二项分布计算 p-value
```
场景：10次猜巧克力，猜对8次

P(X=k) = C(n,k) × p^k × (1-p)^(n-k)

P(X=8) = C(10,8) × 0.5^8 × 0.5^2
        = 45 × 0.5^10
        = 45/1024
        ≈ 4.4%

p-value = P(X≥8)
        = P(8) + P(9) + P(10)
        = 4.4% + 1.0% + 0.1%
        = 5.5%

5.5% > 5% → 不能拒绝H0
```

### ⚠️ 最常见的误解
```
❌ 错误：p=0.03 意思是"H1有97%概率是真的"
✅ 正确：p=0.03 意思是"如果H0为真，出现这种结果的概率只有3%"

p-value 说的是数据的概率，不是假设成立的概率！
```

### α = 0.05 的由来
> Fisher 1925年随手举例，后来全世界把0.05当成金标准！

| 领域 | α标准 | 原因 |
|---|---|---|
| 商业/社会科学 | 0.05 | 误报代价小 |
| 医学临床试验 | 0.01 | 误报会害死人 |
| 粒子物理学 | 0.0000003（5σ） | 发现新粒子需极度确定 |
| 基因研究 | 0.000001 | 同时测百万个基因 |

## 4. Type I & Type II Error

### 混淆矩阵

```
╔══════════════╦══════════════════╦══════════════════╗
║              ║  检验：不显著    ║  检验：显著      ║
╠══════════════╬══════════════════╬══════════════════╣
║ 真实：无效果 ║ ✅ True Negative ║ ❌ Type I Error  ║
║              ║                  ║   误报！False+   ║
╠══════════════╬══════════════════╬══════════════════╣
║ 真实：有效果 ║ ❌ Type II Error ║ ✅ True Positive ║
║              ║   漏报！False-   ║                  ║
╚══════════════╩══════════════════╩══════════════════╝
```

### 定义对比

| | Type I Error | Type II Error |
|---|---|---|
| **别名** | α、误报、False Positive | β、漏报、False Negative |
| **含义** | 没效果，但说有效果 | 有效果，但没发现 |
| **记忆** | 狼来了（其实没狼） | 有狼没发现 |
| **对应ML** | 降低Precision | 降低Recall |
| **控制参数** | α（越小越好） | β（越小越好） |

### 怀孕测试例子（永远记住！）
```
H0：没有怀孕

Type I Error = 没怀孕，测试说怀孕 → 虚惊一场！
Type II Error = 真的怀孕，测试说没怀孕 → 漏掉了！
```

### 两种错误的权衡（跷跷板）
```
减少Type I Error → Type II Error增加
减少Type II Error → Type I Error增加

唯一能同时改善两者的方法：增大样本量！
```

## 5. Statistical Power（统计功效）

### 定义
```
Power = 1 - β
      = 如果真实效果存在，有多大概率能发现它
```

### Power = 0.80 的含义
```
100个真实有效果的实验：
80个：检测到了！p < 0.05 ✅
20个：没检测到，p ≥ 0.05 ❌（Type II Error）

Power = 0.80 不是做80次实验！
是这一次实验有80%概率发现真实效果！
```

### 提高Power的四种方法

| 方法 | 效果 | 代价 |
|---|---|---|
| 增大样本量n | Power↑ | 钱和时间 |
| 增大MDE | Power↑ | 只能检测大效果 |
| 增大α | Power↑ | Type I Error增加 |
| 减小SE | Power↑ | 取决于数据分布 |

> **最推荐：增大样本量，唯一不损害其他指标的方法！**

## 6. β的本质

### β是面积问题！
```
两个分布叠在一起：

H0分布          H1分布
（没效果）       （有效果，右移了）
  ████           ╔══╗████
██████           ║β ║██████
████████         ║  ║████████
                 ╚══╝
         ↑
       临界值（α决定）

β = H1分布落在临界值左边的面积
  = 有真实效果，但实验说不显著的概率

Power = H1分布落在临界值右边的面积
      = 1 - β
```

### 样本量如何影响β
```
样本量小 → 两个分布很近（重叠多）→ β大
样本量大 → 两个分布分得开（重叠少）→ β小 → Power大
```

## 7. 样本量计算

### 影响因素

```
样本量由四个因素决定：

1. α（显著性水平）通常 0.05
2. Power（1-β）通常 0.80
3. 基准转化率（p）当前指标值
4. MDE（最小可检测效应）希望检测的最小差异
```

### 关键规律
```
MDE减半 → 样本量变成4倍！
（样本量与MDE的平方成反比）

例子：
基准转化率5%，MDE=1%，α=0.05，Power=0.80
→ 每组约14,000人

MDE改为0.5%
→ 每组约56,000人（4倍！）
```

### 不同场景的样本量

| 场景 | α | Power | 样本量 |
|---|---|---|---|
| 标准AB Test | 0.05 | 0.80 | 中等 |
| 医学临床试验 | 0.01 | 0.90 | 很大 |
| 探索性研究 | 0.10 | 0.80 | 较小 |
| 快速迭代 | 0.10 | 0.70 | 很小 |

## 8. 标准误 SE

### 公式
```
SE = √(p(1-p)/n)
```

### 转化率对SE的影响

| 转化率p | p(1-p) | SE大小 | 场景 |
|---|---|---|---|
| 50% | 0.25（最大） | 最大 | 便宜贴纸，随机买/不买 |
| 5% | 0.0475 | 中等 | 普通电商运动鞋 |
| 1% | 0.0099 | 最小 | 豪华游艇，几乎没人买 |

### 直觉理解
```
p=50%（贴纸网站）：
= 在嘈杂夜市里听音乐
  背景噪音大，很难听清真实效果

p=1%（游艇网站）：
= 在安静音乐厅里听音乐
  背景噪音小，真实效果清晰可辨
```

### 实际工作中的影响
```
CTR（点击率）通常1%-30%，离50%够远
→ SE不是最大问题

真正的挑战是：
1. MDE太小 → 需要巨大样本量
2. 指标之间矛盾（CTR↑但转化率↓）
3. 用户异质性（新老用户行为差异大）
```

## 9. AA Test

### 是什么
> 实验前，两组用户都看**完全相同**的页面（都是A），跑一个"假实验"

### 为什么做
```
如果AA Test发现显著差异 → 系统有问题！
可能原因：
- 分流不随机
- 数据收集有bug（点击事件重复计数）
- Cookie问题（用户设备切换）

如果AA Test没差异 → 系统正常 → 可以开始AB Test ✅
```

### 类比
> 就像医生手术前先测试手术器械是否正常

## 10. 多重比较问题

### 问题所在
```
同时测试20个功能，每个用α=0.05：

至少出现1个Type I Error的概率
= 1 - (1-0.05)^20
= 1 - 0.95^20
= 约64%！！

从5%膨胀到64%！
```

### 就像彩票
```
买一张彩票中奖概率 = 5%（α = 0.05）
买20张彩票至少中一次 = 64%

多重比较 = 给了自己20次犯Type I Error的机会
```

### 解决方案对比

| 方法 | 原理 | 优点 | 缺点 | 适合场景 |
|---|---|---|---|---|
| **Bonferroni** | α' = α/n | 严格控制整体误报率 | 太保守，漏报多 | 确认性分析 |
| **FDR（BH法）** | 控制误报比例 | 更灵活平衡 | 允许部分误报 | 探索性分析 |
| **提前定主指标** | 只看1个主要指标 | 最实用 | 需要业务判断 | 日常AB Test |

### Bonferroni原理
```
目标：整体Type I Error ≤ 5%

α' = α/n = 0.05/20 = 0.0025

把总误报预算（5%）平均分给每个测试
每个测试只用0.25%的预算
总预算 = 20 × 0.25% = 5% ✅
```

### FDR vs Bonferroni
```
Bonferroni：
"一个假阳性都不能有"
→ 太严格，漏掉很多真实效果

FDR（BH法）：
"找到的结果里，允许5%是假阳性"
→ 更实际，适合探索性分析
→ 基因研究（同时测百万个基因）必用FDR
```

## 11. Novelty Effect & Network Effect

### Novelty Effect（新奇效应）

```
原因：用户因为好奇新变化短期内多点击

识别：把效果按时间分拆
第1周：+20%
第2周：+12%
第3周：+5%  ← 效果在消退 = Novelty Effect

解决：
① 延长实验时间（至少2-4周）
② 只看老用户数据
③ 观察效果随时间变化趋势
```

### Network Effect（网络效应干扰）

```
场景：社交/协作产品
实验组用户 → 发消息给对照组用户
→ 对照组被"污染"！

解决：Cluster Randomization
按地理位置/组织分组：
北京/广州 → 实验组
上海/深圳 → 对照组
确保两组用户几乎不互相影响
```

## 12. 面试陷阱题

### ⚠️ 陷阱1：Peeking Problem

```
❌ 错误做法：
每天看结果，一旦显著就停止

问题：
随机数据自然起伏
允许随时停止 → Type I Error从5%膨胀到30%+！

✅ 正确做法：
提前计算好样本量和运行时长
到达后一次性分析，不中途改变计划
如必须早停：使用Sequential Testing
```

### ⚠️ 陷阱2：统计显著 ≠ 业务价值

```
场景：
p = 0.001（非常显著）
但提升只有 0.01%

需要同时看：
① 统计显著性（p < 0.05）
② 效应大小（差异有实际意义吗？）
③ 置信区间下界是否有价值
④ 实施成本（值得改吗？）
```

### ⚠️ 陷阱3：SRM（Sample Ratio Mismatch）

```
预设50/50分流
实际：实验组48%，对照组52%

→ 分流系统有bug！
→ 结果不可信，必须停止排查

原因可能是：
- 缓存问题
- Bot流量
- 设备兼容问题
```

### ⚠️ 陷阱4：提前停止

```
实验跑3天，p=0.001非常显著
→ 不能提前停止！

原因：
随机游走统计数据自然起伏
提前停止大幅膨胀Type I Error

必须跑满预定样本量
```

### ⚠️ 陷阱5：指标分歧

```
场景：
实验组 CTR ↑8%
实验组 购买转化率 ↓3%

不能只看一个指标！
要看：
① 北极星指标（GMV/Revenue）
② 理解分歧原因
③ 护栏指标（用户满意度）
```

## 13. 统一框架：Type I/II = Precision/Recall

### 你发现的最重要的洞察！

```
同一个混淆矩阵，三个领域的叫法：

统计学          ML/RAG          医学
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Type I Error  = False Positive = 误诊
Type II Error = False Negative = 漏诊
控制α         = 提高Precision  = 提高特异性
提高Power     = 提高Recall     = 提高敏感性
α和Power平衡  = F1 Score       = 综合指标
```

### 同一个跷跷板

```
统计学：
Type I ←————跷跷板————→ Type II
  α                        β

ML/RAG：
Precision ←————跷跷板————→ Recall

医学：
误诊率 ←————跷跷板————→ 漏诊率
```

### 解决方案对应

| 领域 | 同时改善两者的方法 |
|---|---|
| 统计学 | 增大样本量 |
| ML/RAG | 更好的模型+更多高质量数据 |
| 医学 | 更好的检测技术 |

## 14. 完整 A/B Test 设计流程

```
Step 1：明确指标
  主指标：购买转化率
  护栏指标：页面加载速度、用户满意度

Step 2：设置假设
  H0：新功能不影响转化率
  H1：新功能提升转化率

Step 3：计算样本量
  基准转化率 + MDE + α=0.05 + Power=0.80
  → 得出样本量和实验天数

Step 4：AA Test
  先验证系统分流正常，数据收集无bug

Step 5：随机分流50/50
  全程监控SRM（实际比例是否符合预期）

Step 6：运行实验
  不偷看！不提前停！
  注意Novelty Effect（至少跑2周）

Step 7：达到样本量后一次性分析
  计算p-value
  检查置信区间
  注意多重比较问题

Step 8：决策
  统计显著（p<0.05）
  + 效应有业务价值
  + 护栏指标无损害
  → Launch ✅
  否则 → 不Launch，继续迭代
```

## 15. 面试金句（中英文）

### p-value

**中文：**
> "p-value是假设H0为真，观察到当前结果或更极端的概率。本质上是H0分布下右侧尾部的面积。"

**英文：**
> "The p-value is the probability of observing results at least as extreme as ours, assuming the null hypothesis is true."

---

### 结论表达

| 情况 | 中文 | 英文 |
|---|---|---|
| p < 0.05 | 拒绝H0 | We reject the null hypothesis |
| p ≥ 0.05 | 不能拒绝H0 | We fail to reject the null hypothesis |
| ❌ 永远不说 | ~~接受H0~~ | ~~We accept the null hypothesis~~ |

---

### Type I & II Error

**英文：**
> "Type I is a false positive — we think there's an effect when there isn't. Type II is a false negative — we miss a real effect. They're like Precision and Recall — a fundamental trade-off. The only way to improve both simultaneously is to increase sample size."

---

### A/B Test 设计

**英文：**
> "I'd start by defining the primary metric and guardrail metrics, then calculate sample size based on baseline rate, MDE, α=0.05 and Power=0.80. I'd run an AA test first to validate the system, then monitor for SRM throughout. After reaching the predetermined sample size, I'd analyze once — never peek — and make decisions based on statistical significance, effect size, and business value together."

---

### 多重比较

**英文：**
> "When testing multiple hypotheses simultaneously, the family-wise error rate inflates significantly. With 20 tests at α=0.05, the probability of at least one false positive is 1-(0.95)^20 ≈ 64%. We can control this using Bonferroni correction or FDR, or more practically, by pre-specifying a single primary metric."
